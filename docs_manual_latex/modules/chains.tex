\section{Chains}

\subsection{1. Fachsicht}

Im Repository gibt es \textbf{kein} Top-Level-Paket oder Produktmodul mit dem Namen \textbf{chains}. ?Ketten\texttt{} bezeichnet hier \textbf{mehrere unabh\textbackslash\{\}"angige Ablaufketten}, die im Code vorkommen:

\begin{enumerate}
\item \textbf{Kontext-Override-Kette} --- feste Priorit\textbackslash\{\}"at der Kontext-Konfiguration in \texttt{ChatService.\_resolve\_context\_configuration} (\texttt{app/services/chat\_service.py}).
\item \textbf{Explainability `policy\_chain`} --- serialisierte Darstellung der Kontextentscheidung (\texttt{app/context/explainability/context\_explanation\_serializer.py}, Funktion \texttt{\_policy\_chain\_to\_dict}).
\item \textbf{Delegations-Kette} --- Einstieg \texttt{/delegate} \$\textbackslash\{\}rightarrow\$ \texttt{use\_delegation} in \texttt{app/core/commands/chat\_commands.py} \$\textbackslash\{\}rightarrow\$ Agenten-Pipeline in \texttt{app/agents/} (Planner, Delegation, Execution).
\item \textbf{Pipeline-Engine} --- generische Schritt-Ausf\textbackslash\{\}"uhrung in \texttt{app/pipelines/} (\texttt{engine}, \texttt{executors}, \texttt{services/pipeline\_service.py}); Service-Name \texttt{pipeline\_service} in \texttt{docs/SYSTEM\_MAP.md}.
\end{enumerate}
