## Phase 2 — Source of Truth Decisions

| Area | Canonical Source | Notes |
|------|------------------|-------|
| **Application entrypoint** | `main.py` → `run_gui_shell.py` | main.py delegiert direkt |
| **Runtime app source** | `app/` | **app/ is canonical.** src/ is NOT ready for cutover |
| **UI source** | `app/gui/` (shell) + `app/ui/` (widgets) | Both active; gui hosts, ui provides widgets |
| **Assets** | `assets/` | Icons, themes, media. app/resources and app/gui/icons/svg are legacy/duplicate |
| **Help content** | `help/` | **help/ is canonical.** docs/ remains fallback until help/ complete |
| **Project documentation** | `docs/` | README, 00_map, SYSTEM_MAP, TRACE_MAP at root; rest in numbered sections or archive |
| **QA documentation** | `docs/qa/` | QA artifacts, incidents, reports |
| **Generated maps** | `docs/SYSTEM_MAP.md`, `docs/TRACE_MAP.md` | Generated by tools |
| **Archive** | `archive/` | Legacy code, deprecated docs |

### Explicit Answers

- **Is src/ already canonical?** No. src/ is a wrapper only. Full app/ → src/ migration has not happened.
- **Is app/ still the real source tree?** Yes. app/ is canonical.
- **Should assets/ replace app/resources and app/gui/icons now?** assets/ is already canonical for new GUI. app/resources and app/gui/icons/svg are duplicates; keep app/resources for legacy app.main compatibility until legacy is fully archived. Remove app/gui/icons/svg (code uses assets via paths.py).
- **Should help/ already be canonical for user help?** Yes. HelpIndex loads help/ first. docs/ is fallback.
- **Which docs stay in docs root?** README.md, 00_map_of_the_system.md, SYSTEM_MAP.md, TRACE_MAP.md only. Others move to numbered sections or archive.

## Phase 3 — Consolidation Rules

5. **Generated/cache:** __pycache__, .pytest_cache, *.zip artifacts not tracked as documentation.
6. **AUDIT_REPORT.md:** Keep at project root (referenced by app/qa/operations_adapter) or document relocation.
