# QA Documentation – Linux Desktop Chat

**Purpose:** Central hub for QA architecture, governance, reports, and generated artifacts.  
**Audience:** QA leads, developers, architects, CI maintainers.

---

## What is docs/qa?

`docs/qa/` is the **QA documentation system** for Linux Desktop Chat. It contains:

- **Architecture** – How the QA system is designed (Autopilot, Coverage Map, Test Inventory, Incident Replay)
- **Governance** – Test rules, schemas, incident standards, regression catalog
- **Reports** – Human-authored reviews, iteration reports, gap analyses
- **Artifacts** – Machine-generated dashboards (MD), JSON snapshots, CSV (regenerated by scripts)
- **Incidents** – Incident registry, case data, templates
- **Feedback loop** – Trace files from feedback loop runs
- **Config** – Active configuration (Phase 3 configs)
- **Plans** – Active QA plans (chaos, sanitization)
- **History** – Archived phase documents

## How to Run QA Scripts

```bash
# QA Cockpit (generates QA_STATUS)
python scripts/qa/qa_cockpit.py

# Test Inventory
python scripts/qa/build_test_inventory.py

# Coverage Map
python scripts/qa/build_coverage_map.py

# Feedback Loop
python scripts/qa/run_feedback_loop.py --output docs/qa/artifacts/json/FEEDBACK_LOOP_REPORT.json

# Incident Registry
python scripts/qa/incidents/build_registry.py
```
